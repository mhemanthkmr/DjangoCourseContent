Static vs Media Files

Understanding the fundamental difference between static and media files in Django applications.

Static Files:

  • CSS, JavaScript, images, fonts used by your application
  • Same for all users (site-wide assets)
  • Version-controlled with your code
  • Deployed with your application

Media Files:

  • User-uploaded content (profile pictures, blog images, documents)
  • Unique per user/content
  • Stored separately from code
  • Require security considerations

Key Distinction:

  • Static files = application assets (unchanging)
  • Media files = user-generated content (dynamic)

Static Files Configuration

Proper setup for serving static files during development and production deployment.

Settings Configuration:

# settings.py
import os
from pathlib import Path

BASE_DIR = Path(__file__).resolve().parent.parent

# Static files configuration
STATIC_URL = '/static/'
STATIC_ROOT = os.path.join(BASE_DIR, 'staticfiles')  # For production

# Static files directories
STATICFILES_DIRS = [
    os.path.join(BASE_DIR, 'static'),
    os.path.join(BASE_DIR, 'assets'),
]

# Apps with static files
STATICFILES_FINDERS = [
    'django.contrib.staticfiles.finders.FileSystemFinder',
    'django.contrib.staticfiles.finders.AppDirectoriesFinder',
]

Serving Static Files

How Django serves static files in development and production environments.

Development (DEBUG=True):

# urls.py (project level)
from django.conf import settings
from django.conf.urls.static import static
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', include('myapp.urls')),
] + static(settings.STATIC_URL, document_root=settings.STATIC_ROOT)

In Templates:

{% load static %}
<!DOCTYPE html>
<html>
<head>
    <link rel="stylesheet" type="text/css" href="{% static 'css/style.css' %}">
    <script src="{% static 'js/script.js' %}"></script>
</head>
<body>
    <img src="{% static 'images/logo.png' %}" alt="Logo">
</body>
</html>

Production:

  • Run python manage.py collectstatic
  • Configure web server (Nginx, Apache) to serve static files
  • Use CDN for better performance

Media Files Configuration

Setting up user-uploaded media files with proper security and organization.

Settings Configuration:

# settings.py
import os
from pathlib import Path

# Media files configuration
MEDIA_URL = '/media/'
MEDIA_ROOT = os.path.join(BASE_DIR, 'media')

# Security considerations
FILE_UPLOAD_MAX_MEMORY_SIZE = 5242880  # 5MB
DATA_UPLOAD_MAX_MEMORY_SIZE = 5242880  # 5MB

# Allowed file types (optional)
ALLOWED_FILE_EXTENSIONS = ['.jpg', '.jpeg', '.png', '.gif']

URL Configuration:

# urls.py (project level)
from django.conf import settings
from django.conf.urls.static import static
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', include('myapp.urls')),
] + static(settings.MEDIA_URL, document_root=settings.MEDIA_ROOT)

Security Warning:

  • Never serve media files in production using Django
  • Use separate storage (AWS S3, etc.) for production
  • Validate file types and sizes

File Upload in Models

Implementing file upload functionality in Django models with proper validation and storage.

Model with File Fields:

# models.py
from django.db import models
from django.contrib.auth.models import User
import os

def upload_to(instance, filename):
    """Generate upload path for user-uploaded files"""
    return f'blog_images/{instance.author.username}/{filename}'

class Post(models.Model):
    title = models.CharField(max_length=200)
    content = models.TextField()
    cover_image = models.ImageField(
        upload_to=upload_to,
        blank=True,
        null=True,
        help_text="Upload a cover image for this post"
    )
    author = models.ForeignKey(User, on_delete=models.CASCADE)
    created_at = models.DateTimeField(auto_now_add=True)
    
    def delete(self, *args, **kwargs):
        """Delete file when model instance is deleted"""
        if self.cover_image:
            if os.path.isfile(self.cover_image.path):
                os.remove(self.cover_image.path)
        super().delete(*args, **kwargs)

File Validation:

from django.core.validators import FileExtensionValidator
from django.core.exceptions import ValidationError

def validate_image_size(value):
    """Validate file size (max 5MB)"""
    filesize = value.size
    if filesize > 5242880:  # 5MB
        raise ValidationError("Max file size is 5MB")

class Post(models.Model):
    cover_image = models.ImageField(
        upload_to=upload_to,
        blank=True,
        null=True,
        validators=[
            FileExtensionValidator(allowed_extensions=['jpg', 'jpeg', 'png']),
            validate_image_size
        ]
    )

File Upload Forms & Views

Creating forms and views that handle file uploads with proper validation and error handling.

Model Form with File Upload:

# forms.py
from django import forms
from .models import Post

class PostForm(forms.ModelForm):
    class Meta:
        model = Post
        fields = ['title', 'content', 'cover_image']
        widgets = {
            'title': forms.TextInput(attrs={'class': 'form-control'}),
            'content': forms.Textarea(attrs={'class': 'form-control', 'rows': 5}),
            'cover_image': forms.FileInput(attrs={'class': 'form-control'})
        }

View Handling File Upload:

# views.py
from django.shortcuts import render, redirect
from django.contrib import messages
from .forms import PostForm

def create_post(request):
    if request.method == 'POST':
        form = PostForm(request.POST, request.FILES)
        if form.is_valid():
            post = form.save(commit=False)
            post.author = request.user
            post.save()
            messages.success(request, 'Post created successfully!')
            return redirect('post_detail', pk=post.pk)
    else:
        form = PostForm()
    
    return render(request, 'blog/create_post.html', {'form': form})

Template with File Upload:

{% load static %}
<form method="post" enctype="multipart/form-data">
    {% csrf_token %}

    <button type="submit">Create Post</button>
</form>

<!-- Display uploaded image -->
    <img src="{{ post.cover_image.url }}" alt="Cover Image" class="img-fluid">

Production Considerations

Best practices for handling static and media files in production environments.

Cloud Storage (AWS S3):

# settings.py (production)
import os

# AWS S3 Configuration
AWS_ACCESS_KEY_ID = os.environ.get('AWS_ACCESS_KEY_ID')
AWS_SECRET_ACCESS_KEY = os.environ.get('AWS_SECRET_ACCESS_KEY')
AWS_STORAGE_BUCKET_NAME = 'your-bucket-name'
AWS_S3_REGION_NAME = 'us-east-1'
AWS_S3_CUSTOM_DOMAIN = f'{AWS_STORAGE_BUCKET_NAME}.s3.amazonaws.com'

# Static files on S3
STATICFILES_STORAGE = 'storages.backends.s3boto3.S3Boto3Storage'

# Media files on S3
DEFAULT_FILE_STORAGE = 'storages.backends.s3boto3.S3Boto3Storage'
AWS_DEFAULT_ACL = 'public-read'

Performance Optimization:

  • Use CDN for static files (CloudFront, Cloudflare)
  • Implement image optimization and compression
  • Set proper cache headers
  • Use WebP format for images
  • Lazy loading for images

Security Best Practices:

  • Validate file types and sizes
  • Scan uploaded files for malware
  • Store files in private buckets when needed
  • Use signed URLs for private content
