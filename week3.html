Introduction to Databases and ORM

What is a Database?

A structured set of data held in a computer, especially one that is accessible in various ways.

What is ORM?

Object-Relational Mapping (ORM) is a technique that lets you query and manipulate data from a database using an object-oriented paradigm.

Django ORM allows you to interact with your database like you would with SQL. It abstracts the database layer and lets you work with Python objects instead of writing raw SQL queries.

Benefits of ORM:

  • Database abstraction (switch between SQLite, PostgreSQL, MySQL easily)
  • Protection against SQL injection attacks
  • Write Python code instead of SQL
  • Automatic schema generation and migrations

Defining Models (models.py)

In Django, each model is a Python class that subclasses django.db.models.Model.

Each model represents a database table, and each attribute of the model represents a database field.

Creating a Model:

# models.py
from django.db import models

class Author(models.Model):
    name = models.CharField(max_length=100)
    email = models.EmailField()

class Post(models.Model):
    title = models.CharField(max_length=200)
    content = models.TextField()
    created_at = models.DateTimeField(auto_now_add=True)

Key Points:

  • File location: myapp/models.py
  • Each model becomes a database table
  • Django automatically adds an id field as primary key
  • After creating models, run migrations to create database tables

Field Types

Django provides many built-in field types to represent different data types in your database.

Common Field Types:

# Character fields
models.CharField(max_length=255)          # Short text
models.TextField()                        # Long text

# Numeric fields
models.IntegerField()
models.FloatField()
models.DecimalField(max_digits=5, decimal_places=2)

# Date/Time fields
models.DateField()
models.DateTimeField()
models.TimeField()

# Boolean fields
models.BooleanField()

# Special fields
models.EmailField()
models.URLField()
models.FileField()
models.ImageField()

Field Options:

  • null=True - Allow NULL values in database
  • blank=True - Allow empty values in forms
  • default=value - Set default value
  • unique=True - Ensure field value is unique
  • choices=CHOICES - Limit choices to specific options

Relationships

Django provides three types of relationships to connect models:

ForeignKey (Many-to-One)

Many objects of one model can be related to one object of another model.

class Post(models.Model):
    author = models.ForeignKey(Author, on_delete=models.CASCADE)

on_delete options: CASCADE, PROTECT, SET_NULL, SET_DEFAULT

OneToOneField (One-to-One)

Each object of one model is linked to exactly one object of another model.

class AuthorProfile(models.Model):
    author = models.OneToOneField(Author, on_delete=models.CASCADE)

ManyToManyField (Many-to-Many)

Many objects of one model can be related to many objects of another model.

class Post(models.Model):
    tags = models.ManyToManyField(Tag)

Migrations (makemigrations, migrate)

Migrations are Django's way of propagating changes you make to your models into your database schema.

Migration Workflow:

  1. Create or modify models in models.py
  2. Run python manage.py makemigrations to create migration files
  3. Run python manage.py migrate to apply migrations to database

Common Migration Commands:

# Create new migration files
python manage.py makemigrations

# Apply migrations to database
python manage.py migrate

# Show migration status
python manage.py showmigrations

# Create empty migration file
python manage.py makemigrations --empty myapp

Important Notes:

Important Notes:

  • Migration files are stored in myapp/migrations/ directory
  • Never edit migration files manually unless you know what you're doing
  • Always run migrations after changing models

Django Admin Interface

Django provides a ready-to-use admin interface for managing your application data.

Setting Up Admin:

  1. Create superuser: python manage.py createsuperuser
  2. Access admin at http://127.0.0.1:8000/admin/
  3. Register your models in admin.py

Registering Models:

# admin.py
from django.contrib import admin
from .models import Author, Post

admin.site.register(Author)
admin.site.register(Post)

Customizing Admin:

@admin.register(Post)
class PostAdmin(admin.ModelAdmin):
    list_display = ('title', 'author', 'created_at')
    list_filter = ('created_at', 'author')
    search_fields = ('title', 'content')

Lab: Build a Blog Model

Requirements:

  • Create Author model with name and email
  • Create Post model with title, content, and publication date
  • Establish relationship between Author and Post (one-to-many)
  • Register models in Django admin

Implementation:

# models.py
from django.db import models

class Author(models.Model):
    name = models.CharField(max_length=100)
    email = models.EmailField(unique=True)
    
    def __str__(self):
        return self.name

class Post(models.Model):
    title = models.CharField(max_length=200)
    content = models.TextField()
    author = models.ForeignKey(Author, on_delete=models.CASCADE)
    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)
    
    def __str__(self):
        return self.title

Lab Steps:

  1. Create new Django app: python manage.py startapp blog
  2. Define models in blog/models.py
  3. Add 'blog' to INSTALLED_APPS in settings.py
  4. Create and apply migrations
  5. Register models in blog/admin.py
  6. Create superuser and test in admin interface
