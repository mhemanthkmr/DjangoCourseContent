Week 6: Authentication & User Management

Securing Django applications with built-in authentication and user management

Built-in Django Auth System

Django provides a robust, secure authentication system out-of-the-box with models, views, and forms for user management.

Core Components:

  • User model with username, email, password, and permissions
  • Authentication views for login/logout/password management
  • Authentication middleware for request.user
  • Permission system for fine-grained access control

User Model Structure:

from django.contrib.auth.models import User

# Default fields include:
# - username (unique)
# - email
# - password (hashed)
# - first_name, last_name
# - is_active, is_staff, is_superuser
# - date_joined, last_login

Key Insight:

  • Never store passwords in plain text - Django handles hashing automatically
  • Use request.user in views to access the current user
2

Login, Logout & Registration

Django provides built-in views for authentication workflows, but custom registration requires additional setup.

Using Built-in Auth Views:

# urls.py
from django.contrib.auth import views as auth_views

urlpatterns = [
    path('login/', auth_views.LoginView.as_view(), name='login'),
    path('logout/', auth_views.LogoutView.as_view(), name='logout'),
    path('password_change/', auth_views.PasswordChangeView.as_view(), name='password_change'),
]

Custom Registration View:

# views.py
from django.contrib.auth.forms import UserCreationForm
from django.shortcuts import render, redirect

def register(request):
    if request.method == 'POST':
        form = UserCreationForm(request.POST)
        if form.is_valid():
            form.save()
            return redirect('login')
    else:
        form = UserCreationForm()
    return render(request, 'registration/register.html', {'form': form})
  • Create registration/login.html template for login
  • Use UserCreationForm for registration
  • Set LOGIN_REDIRECT_URL in settings.py

User Permissions & Groups

Django's permission system allows fine-grained control over what users can do in your application.

Permission Types:

  • Model-level permissions - add, change, delete, view
  • Object-level permissions - require custom implementation
  • Groups - collections of permissions assigned to users

Checking Permissions:

# In views
from django.contrib.auth.decorators import permission_required

@permission_required('blog.add_post')
def create_post(request):
    # Only users with 'add_post' permission can access

# In templates
{% if user.has_perm('blog.add_post') %}
    Create Post
{% endif %}

# In models
class Post(models.Model):
    class Meta:
        permissions = [
            ("publish_post", "Can publish posts"),
        ]

Security Note:

  • Always validate permissions in views, not just templates
  • Use @login_required decorator for authenticated views

Password Management

Django provides secure password handling with hashing, validation, and reset functionality.

Password Security Features:

  • PBKDF2 with SHA256 by default (configurable)
  • Password validators for complexity requirements
  • Password reset via email with token-based links
  • Password change with old password verification

Password Validators Configuration:

# settings.py
AUTH_PASSWORD_VALIDATORS = [
    {
        'NAME': 'django.contrib.auth.password_validation.UserAttributeSimilarityValidator',
    },
    {
        'NAME': 'django.contrib.auth.password_validation.MinimumLengthValidator',
        'OPTIONS': {'min_length': 8}
    },
    {
        'NAME': 'django.contrib.auth.password_validation.CommonPasswordValidator',
    },
    {
        'NAME': 'django.contrib.auth.password_validation.NumericPasswordValidator',
    },
]

Best Practices:

  • Never implement custom password hashing - use Django's built-in system
  • Enable password validators to enforce strong passwords
  • Use HTTPS in production to protect password transmission

Lab: Implement Authentication

Requirements:

  • Set up login/logout functionality using Django's built-in views
  • Create a registration page using UserCreationForm
  • Restrict post creation to authenticated users only
  • Add navigation links that change based on authentication status

Solution Implementation:

# views.py
from django.contrib.auth.decorators import login_required
from django.shortcuts import render, redirect
from .forms import PostForm

@login_required
def create_post(request):
    if request.method == 'POST':
        form = PostForm(request.POST)
        if form.is_valid():
            post = form.save(commit=False)
            post.author = request.user
            post.save()
            return redirect('post_list')
    else:
        form = PostForm()
    return render(request, 'blog/create_post.html', {'form': form})
<!-- base.html -->
{% if user.is_authenticated %}
    <p>Hello, {{ user.username }}!</p>
    <a href="{% url 'logout' %}">Logout</a>
    <a href="{% url 'create_post' %}">Create Post</a>
{% else %}
    <a href="{% url 'login' %}">Login</a>
    <a href="{% url 'register' %}">Register</a>
{% endif %}
