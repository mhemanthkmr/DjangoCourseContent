Registering Models in the Admin

By default, models are not visible in the Django admin. You must explicitly register them to manage data through the admin interface.

Basic Registration:

# admin.py
from django.contrib import admin
from .models import Author, Post

admin.site.register(Author)
admin.site.register(Post)

Best Practices:

  • Only register models that need admin access
  • Use @admin.register decorator for cleaner syntax
  • Define __str__() methods in models for readable object names
2

Customizing ModelAdmin

Use ModelAdmin to tailor how models appear and behave in the admin.

Key Customization Options:

  • list_display – Columns shown in the list view
  • search_fields – Enable full-text search
  • list_filter – Add sidebar filters
  • date_hierarchy – Drill-down by date
@admin.register(Post)
class PostAdmin(admin.ModelAdmin):
    list_display = ('title', 'author', 'created_at', 'is_published')
    list_filter = ('created_at', 'author', 'is_published')
    search_fields = ('title', 'content')
    date_hierarchy = 'created_at'
3

Inline Editing for Related Data

Edit related objects directly from a parent model’s admin page using inline forms.

TabularInline vs StackedInline:

  • TabularInline – Compact table-style layout (good for many fields)
  • StackedInline – Vertical, form-like layout (better for few fields)
class CommentInline(admin.TabularInline):
    model = Comment
    extra = 1

@admin.register(Post)
class PostAdmin(admin.ModelAdmin):
    inlines = [CommentInline]

Notes:

Important:

  • Set extra to control number of empty forms
  • Use readonly_fields or can_delete=False for safety
4

Custom Admin Actions

Admin actions allow bulk operations on selected objects (e.g., publish multiple posts at once).

Creating a Custom Action:

def publish_posts(modeladmin, request, queryset):
    queryset.update(is_published=True)
publish_posts.short_description = "Publish selected posts"

@admin.register(Post)
class PostAdmin(admin.ModelAdmin):
    actions = [publish_posts]
    list_display = ('title', 'is_published')

Advanced Tips:

  • Use queryset to safely update many records
  • Add permissions checks inside actions
  • Return HttpResponse for complex workflows

Admin Permissions & Field-Level Security

Django admin integrates with Django’s built-in authentication and permissions system.

Key Concepts:

  • Users can be assigned to Groups with specific permissions
  • Each model has auto-generated add, change, delete, and view permissions
  • Restrict fields per user using get_form() or get_fieldsets()
@admin.register(Post)
class PostAdmin(admin.ModelAdmin):
    def get_form(self, request, obj=None, **kwargs):
        form = super().get_form(request, obj, **kwargs)
        if not request.user.is_superuser:
            form.base_fields.pop('author', None)
        return form

Security Reminder:

  • Never expose sensitive fields (e.g., passwords) in admin
  • Test permissions with non-superuser accounts
  • Use readonly_fields for audit trails

Admin Theming & Usability Tips

While Django admin is functional by default, small tweaks greatly improve user experience.

Simple Theming:

  • Override admin templates in templates/admin/
  • Use packages like django-jazzmin or django-grappelli for full UI overhauls

Usability Enhancements:

  • Set autocomplete_fields for ForeignKey lookups
  • Use prepopulated_fields for slugs (e.g., from title)
  • Add helpful help_text in model fields
  • Organize fields with fieldsets
@admin.register(Post)
class PostAdmin(admin.ModelAdmin):
    prepopulated_fields = {"slug": ("title",)}
    fieldsets = (
        ('Content', {'fields': ('title', 'slug', 'content')}),
        ('Metadata', {'fields': ('author', 'is_published'), 'classes': ('collapse',)}),
    )

Lab: Customize Blog Admin

Requirements:

  • Register Author and Post models
  • Add inline editing for Comments or Tags
  • Implement a “Publish Selected Posts” admin action
  • Hide sensitive fields (e.g., author) from non-superusers

Example Implementation:

# admin.py
from django.contrib import admin
from .models import Author, Post, Comment

class CommentInline(admin.StackedInline):
    model = Comment
    extra = 0

def publish_posts(modeladmin, request, queryset):
    updated = queryset.update(is_published=True)
    modeladmin.message_user(request, f"{updated} posts published.")
publish_posts.short_description = "Publish selected posts"

@admin.register(Post)
class PostAdmin(admin.ModelAdmin):
    list_display = ('title', 'author', 'is_published', 'created_at')
    list_filter = ('is_published', 'created_at')
    search_fields = ('title', 'content')
    actions = [publish_posts]
    inlines = [CommentInline]
    
    def get_form(self, request, obj=None, **kwargs):
        form = super().get_form(request, obj, **kwargs)
        if not request.user.is_superuser:
            form.base_fields.pop('author', None)
        return form

admin.site.register(Author)

  2. Edit blog/admin.py to include customizations
  3. Create a staff user (non-superuser) to test field restrictions
  4. Verify inline comments, search, filters, and publish action work
  5. Test with both superuser and staff accounts
