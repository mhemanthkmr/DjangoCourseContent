HTML Forms vs. Django Forms

While raw HTML forms work, Django forms provide validation, security, and maintainability benefits.

HTML Form Example:

<form method="post">
    <input type="text" name="title" required>
    <textarea name="content"></textarea>
    <button type="submit">Submit</button>
</form>

Django Form Advantages:

  • Automatic CSRF protection
  • Built-in validation and error handling
  • Easy rendering with {{ form.as_p }}
  • Type-safe data handling with cleaned_data

Key Insight:

  • Django forms handle both rendering and processing
  • Always use Django forms for user input in production
2

Creating Forms: Form vs. ModelForm

Django provides two main form types: forms.Form for custom logic and forms.ModelForm for database models.

Custom Form (forms.Form):

# forms.py
from django import forms

class ContactForm(forms.Form):
    name = forms.CharField(max_length=100)
    email = forms.EmailField()
    message = forms.CharField(widget=forms.Textarea)

Model-Based Form (ModelForm):

from django.forms import ModelForm
from .models import Post

class PostForm(ModelForm):
    class Meta:
        model = Post
        fields = ['title', 'content', 'author']
  • Use ModelForm for CRUD operations on models
  • Use Form for non-model data (e.g., contact forms)
  • Customize fields with widgets and validators

Handling GET/POST Requests

Django views must distinguish between initial form display (GET) and form submission (POST).

View Pattern:

from django.shortcuts import render, redirect
from .forms import PostForm

def create_post(request):
    if request.method == 'POST':
        form = PostForm(request.POST)
        if form.is_valid():
            form.save()
            return redirect('post_list')
    else:
        form = PostForm()  # Empty form for GET
    
    return render(request, 'blog/create_post.html', {'form': form})

Template Example:

<!-- create_post.html -->
<form method="post">
    {% csrf_token %}
    {{ form.as_p }}
    <button type="submit">Create Post</button>
</form>

Security Note:

  • Always include {% csrf_token %} in POST forms
  • Never process form data without form.is_valid()

Form Validation & Error Handling

Django provides multiple validation layers: field-level, form-level, and model-level.

Validation Types:

  • Field validators – e.g., MinLengthValidator
  • clean_fieldname() – Custom field validation
  • clean() – Cross-field validation
  • Model clean() – Database-level constraints

Custom Validation Example:

class PostForm(ModelForm):
    class Meta:
        model = Post
        fields = ['title', 'content']
    
    def clean_title(self):
        title = self.cleaned_data['title']
        if len(title) < 5:
            raise forms.ValidationError("Title must be at least 5 characters.")
        return title
    
    def clean(self):
        cleaned_data = super().clean()
        title = cleaned_data.get('title')
        content = cleaned_data.get('content')
        if title and content and title.lower() in content.lower():
            raise forms.ValidationError("Title cannot appear in content.")

Best Practices:

  • Always return cleaned data from validation methods
  • Use {{ form.non_field_errors }} in templates for form-wide errors
  • Test validation with invalid inputs

Solution Implementation:

# forms.py
from django import forms
from .models import Post

class PostForm(forms.ModelForm):
    class Meta:
        model = Post
        fields = ['title', 'content', 'author']
    
    def clean_title(self):
        title = self.cleaned_data['title']
        if len(title) < 5:
            raise forms.ValidationError("Title must be at least 5 characters.")
        return title
    
    def clean_content(self):
        content = self.cleaned_data['content']
        if len(content) < 20:
            raise forms.ValidationError("Content must be at least 20 characters.")
        return content
