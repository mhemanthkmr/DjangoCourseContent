What is Django? Why Use It?

Django is a high-level Python web framework that encourages rapid development and clean, pragmatic design.

Key Features:

  • Batteries-included philosophy – comes with ORM, admin, auth, and more
  • Secure by default – protects against XSS, CSRF, SQL injection
  • Scalable and used by companies like Instagram, Mozilla, and Pinterest
  • Excellent documentation and large community

Why Django?

  • Reduces time-to-market with built-in tools
  • Enforces clean, maintainable code structure
  • Great for MVPs, prototypes, and full-scale applications
2

MVC vs. MTV Architecture

Django follows the MTV (Model-Template-View) pattern, which is Django’s interpretation of MVC.

Comparison:

  • Model = Data layer (same as MVC)
  • Template = Presentation layer (equivalent to View in MVC)
  • View = Business logic layer (equivalent to Controller in MVC)

Model

Database logic

ORM

View

Request handling

Business logic

Template

HTML rendering

Dynamic content

This separation promotes modularity and testability.

Setting Up the Development Environment

Before coding, set up a clean, isolated environment.

Step-by-Step Setup:

  1. Install Python (≥ 3.8)
  2. Use venv to create a virtual environment: 
    python -m venv myenv
source myenv/bin/activate  # Linux/macOS
myenv\Scripts\activate     # Windows
  3. Install Django: 
    pip install django

Best Practice:

  • Always use a virtual environment per project
  • Pin dependencies with pip freeze > requirements.txt

Creating Your First Django Project and App

A Django project is the entire web application. An app is a modular component (e.g., blog, user auth).

Commands:

# Create project
django-admin startproject mysite

# Create app inside project
cd mysite
python manage.py startapp hello

Project Structure:

  • mysite/ – Project root
  • manage.py – Command-line utility
  • mysite/settings.py – Configuration
  • hello/ – Your app directory
5

Understanding manage.py, settings.py & Structure

manage.py

Command-line utility for administrative tasks:

  • runserver – Start dev server
  • makemigrations, migrate
  • createsuperuser

settings.py

Central configuration file for your project:

  • INSTALLED_APPS – List of active apps
  • DATABASES – Database config (SQLite by default)
  • DEBUG – Enable dev mode (never in production!)
  • ALLOWED_HOSTS – Security setting for deployment

App Structure:

  • views.py – Request/response logic
  • models.py – Data models
  • urls.py – URL routing (create if missing)
  • templates/ – HTML files

Lab: Build a “Hello, World!” Django App

Requirements:

  • Create a new Django project
  • Create a new app called hello
  • Display “Hello, World!” in the browser

Implementation Steps:

# hello/views.py
from django.http import HttpResponse

def home(request):
    return HttpResponse("Hello, World!")
# hello/urls.py
from django.urls import path
from . import views

urlpatterns = [
    path('', views.home, name='home'),
]
# mysite/urls.py
from django.contrib import admin
from django.urls import include, path

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', include('hello.urls')),
]

Lab Steps:

  1. Create project and app
  2. Add 'hello' to INSTALLED_APPS
  3. Implement views, URLs as above
  4. Run python manage.py runserver
  5. Visit http://127.0.0.1:8000
